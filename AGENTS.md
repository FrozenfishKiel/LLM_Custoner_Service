# Repo-local Codex Working Agreement

本项目是电商智能客服系统。任何 Codex/agent 在本仓库工作时，必须优先遵守以下规则。

## 完整业务测试优先

- 不能只用“写一段测一段”的局部测试作为完成依据。
- 涉及账号、授权、聊天、LLM、Flow、Action、数据库、前端、部署或监控的改动，必须阅读并遵守：
  - `docs/COMPLETE_BUSINESS_TEST_CHECKLIST.md`
- 每次完成声明前，必须说明跑过哪些完整业务链路测试，以及量化指标是否回退。
- 写操作必须验证数据库最终状态，不得只看回复文本。
- LLM/Flow 相关改动必须覆盖真实 LLM 评测或说明为什么当前无法运行。
- 如果浏览器 E2E、压测、安全测试或监控验证因为环境缺失无法运行，必须明确记录未覆盖风险，不能把任务说成上线就绪。

## 中文文档

- 项目规格、报告、测试清单、交接说明和用户可见文档必须使用中文。
- 测试报告必须包含命令、退出码、通过/失败数量、量化指标和剩余风险。

## 安全与配置

- 不得提交 `.env`、`.venv`、缓存、密钥、Cookie、Session、CSRF、数据库密码或 LLM API key。
- 不得在日志、测试输出、报告或提交信息中打印密钥值。

## 多 agent 协作

- 多 agent 并行时，必须先明确文件边界。
- 只读审查 agent 不得修改文件。
- 最终集成、验证、提交由主 agent 统一收口。
